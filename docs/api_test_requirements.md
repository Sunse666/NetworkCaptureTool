# 接口捕获软件需求文档

## 1. 项目定位

接口捕获软件是一款面向接口自动化测试准备阶段的桌面工具。用户通过软件启动受控浏览器，在目标网站中正常操作后，软件自动采集浏览器内所有标签页产生的网络请求，并把请求参数、响应、Cookie、令牌、cURL 和批量导出文件结构化展示。

当前版本重点解决三件事：

- 从浏览器真实操作中快速获得接口请求。
- 清理浏览器噪音参数，保留接口调试和自动化脚本需要的关键信息。
- 批量导出为 Postman、OpenAPI、cURL 和 Apifox 可导入文件。

## 2. 当前技术形态

- 后端采用 Python + FastAPI。
- 前端采用 Vue。
- 桌面版默认使用 SQLite，数据库文件位于运行目录 `runtime/network_capture_studio.db`。
- 开发或团队部署可切换 MySQL，表结构见 `database/schema.sql`。
- 桌面版内置 `chromedriver.exe`，仍要求用户机器安装兼容版本 Chrome。
- 打包后的 exe 使用控制台启动，终端显示访问地址，用户自行用浏览器打开。

## 3. 核心流程

1. 用户启动 `NetworkCaptureTool.exe`。
2. 终端显示本地访问地址，默认 `http://127.0.0.1:8710/`。
3. 用户在网页中输入目标网址并点击“启动浏览器”。
4. 软件启动受控 Chrome，并访问目标网址。
5. 用户在受控浏览器内登录、跳转、点击、打开新标签页。
6. 后端通过 Chrome DevTools Protocol 采集网络请求。
7. 前端轮询同步请求，并以列表和详情弹窗展示。
8. 用户可复制 cURL、复制 Body JSON、查看 Cookie/令牌、批量导出接口文件。

## 4. 功能需求

### 4.1 浏览器启动与登录态

- 支持输入任意目标网址。
- 未输入协议时自动补全 `https://`。
- 启动受控 Chrome 后不等待目标页面完全加载，避免慢页面卡死。
- 用户关闭受控浏览器后，再次启动应能自动重建会话。
- 受控 Chrome 使用独立用户目录保存 Cookie、LocalStorage、SessionStorage。
- 支持“重置登录态”，用于清理过期 Cookie 或目标站反复提示 token 无效的情况。

### 4.2 请求采集

- 采集受控浏览器中所有标签页的网络请求。
- 支持用户在受控浏览器内手动打开新标签页，新标签页请求也应采集。
- 采集 GET、POST、PUT、PATCH、DELETE 等方法。
- 自动识别接口类请求与静态资源请求。
- 同一批次内相同 `method + domain + path` 的重复请求只保留最新一条。
- 每次启动新采集批次时，历史请求 `NEW` 标记自动清除。
- 当前批次新增请求展示 `NEW` 标记。
- 用户点击请求详情后，该请求的 `NEW` 标记立即消失。

### 4.3 请求列表

- 支持按请求方法筛选。
- 支持只看接口请求。
- 支持按 URL 或状态码关键词搜索。
- 支持滑动加载，避免大量请求一次性渲染造成卡顿。
- 支持选择本页请求、单个勾选、单个取消。
- 支持一键清空当前采集请求，不删除历史批次。

### 4.4 请求详情

- 点击请求后以弹窗详情显示，不挤占左侧列表区域。
- 参数视图展示 Query、Body、Headers、Cookie、令牌、响应和 cURL。
- Body 参数支持单独复制为标准 JSON。
- Cookie、Header、令牌等信息完整展示，不做默认脱敏。
- 对二进制、压缩体、protobuf 等疑似不可读内容给出友好占位，避免参数视图乱码。

### 4.5 cURL 与批量导出

- 支持完整 cURL 和精简 cURL。
- 精简 cURL 去掉常见浏览器噪音 Header，例如 `sec-fetch-*`、`sec-ch-ua-*`、`user-agent` 等。
- 支持选中多条请求批量导出。
- 当前支持导出格式：
- Postman Collection v2.1 JSON。
- OpenAPI 3.0 JSON。
- 批量 cURL 文本。
- Apifox 项目 JSON。

### 4.6 背景与界面设置

- 支持预设背景切换。
- 支持选择本机图片作为自定义背景。
- 支持调节背景模糊度和遮罩透明度。
- 前端页面不展示技术栈信息。
- 界面要求轻量、流畅、点击有反馈，保持毛玻璃风格但避免遮挡背景。

### 4.7 历史与日志

- 支持查看历史采集批次。
- 支持按目标网址关键词过滤历史批次。
- 记录用户关键操作日志，例如启动采集、重置登录态、查看详情、清空当前采集、批量导出。
- 日志采用滚动文件，默认保留 5 个文件，每个文件最大 5MB。

## 5. 已取消或暂缓功能

- 暂停采集按钮已移除。
- 时间戳分类视图已移除。
- 默认脱敏展示已移除。
- 单独 Postman 导出接口已移除，统一使用批量导出接口。
- Mock 响应生成暂缓。
- 自动生成断言暂缓。

## 6. 异常提示

- 目标网址为空时提示用户输入网址。
- 目标网址非法时提示检查网址格式。
- 浏览器启动失败时提示检查 Chrome、内置驱动、端口占用或目标站访问情况。
- 浏览器窗口被用户关闭时提示重新启动浏览器。
- 数据库未就绪时根据 SQLite 或 MySQL 场景给出不同提示。
- 请求超时时提示用户重新尝试或检查浏览器状态。

## 7. 非功能需求

- 代码结构保持低耦合，按 API、Service、Repository、Model、Schema、Utils 分层。
- 后续新增导出格式时优先扩展 `BatchExportService`。
- 后续新增设置项时优先扩展 Settings 模块，不直接散落在 UI 组件中。
- 代码注释使用中文，重点解释业务意图和复杂流程。
- 桌面版应支持无 Python、Node、npm、MySQL 环境的普通电脑运行。
- 普通电脑仍需安装兼容版本 Chrome；如需完全免环境，可后续内置 Chrome for Testing。

## 8. 验收标准

- 打包 exe 后，终端显示本地访问地址，且不会自动打开网页。
- 在无 Python、Node、npm、MySQL 命令可见的模拟环境下，exe 可启动并返回健康检查。
- 输入百度或其它公网网址时，能启动受控浏览器。
- 用户在受控浏览器内操作时，请求列表能同步新增请求。
- 重复路径请求只显示最新一条。
- 点击 `NEW` 请求后标记立即消失。
- Body JSON 可单独复制。
- 批量导出的 Postman、OpenAPI、cURL、Apifox 文件可下载。
- 后端测试通过，前端可构建，桌面版可重新打包。
